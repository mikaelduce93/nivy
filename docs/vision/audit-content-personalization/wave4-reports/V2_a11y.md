# V2 — Accessibility (WCAG 2.1 AA) Audit (TICKET-047)

**Sub-agent**: V2 (Wave 4 validators) — READ-ONLY
**Scope**: 30 critical files across teen / parent / partner / admin surfaces + cross-cutting components
**Method**: Manual static review against WCAG 2.1 AA — semantic HTML, ARIA, contrast (from class names), focus, headings, label associations, alt text. No eslint-plugin-jsx-a11y was run (write-scope forbidden).
**Verdicts**: PASS / PARTIAL / FAIL with top-3 issues + estimated 0–100 WCAG score per page.

## 2. Aggregate top-10 patterns to fix

Ranked by frequency × severity:

1. **Icon-only buttons without `aria-label`** — appears in scanner (X close), wallet (`MoreVertical`), friends MessageCircle/MoreVertical, parent dashboard "Historique"/"Sécurité" Buttons (which are also non-functional). **Severity: HIGH (WCAG 4.1.2)**. Estimate ~25+ instances site-wide.

2. **Missing `role="alert"` on error banners** — teen/mentors error, teen/rides/request error, partner/scanner error card, several admin pages. Errors silently appear for SR users. **Severity: HIGH (WCAG 4.1.3)**.

3. **`<div onClick>` antipattern** — most egregious in `app/partner/scanner/page.tsx` offer-selection rows. Also visible (in spirit) in some card-link wrappers across teen surfaces. **Severity: CRITICAL (WCAG 2.1.1, 4.1.2)**.

4. **Filter / tab buttons without `aria-pressed` or `role="tab"`** — wallet (category filters), friends (tab triggers), food/menu-cart-client (filter chips), defis-physiques category chips. Entire UI pattern repeated. **Severity: HIGH (WCAG 4.1.2)**.

5. **Form `<select>` / `<input>` without `<label htmlFor>`** — teen/food filters (sub_category, tag), teen/mentors filter form (labels NOT linked), partner/scanner manual code input, friends search. **Severity: CRITICAL (WCAG 1.3.1, 3.3.2)**.

6. **Low-contrast `text-zinc-500/600` on dark backgrounds** — pervasive. Wallet sub-headers, parent dashboard "DH" suffix, partner dashboard quick-links, quests hub `text-[10px] text-zinc-500`. ~3.5:1 ratio fails AA 4.5:1. **Severity: HIGH (WCAG 1.4.3)**.

7. **Tiny text (`text-[10px]`)** used for navigation labels (mobile-dock) and primary metadata across hubs. Below the AA target of 12px+ for body text and breaks reflow at 200% zoom (WCAG 1.4.10). **Severity: MEDIUM**.

8. **Missing motion-safe gating on parallax / pulse / continuous animations** — partner dashboard background ParallaxLayer, parent dashboard parallax. `prefers-reduced-motion` respected only inside button.tsx and a few targeted spots. **Severity: HIGH (WCAG 2.3.3)**.

9. **Heading hierarchy gaps** — teen/food has only `<h1>` then jumps to plain `<div>`s; menu-cart-client has no `<h3>` per item; admin proofs feed-post bodies are `<div>` content. Adds scan-skip difficulty for AT users. **Severity: MEDIUM (WCAG 1.3.1, 2.4.6)**.

10. **Status / state communicated via color or icon shape only** — friends online-dot (no sr-only label), wallet "current tier" badge, ride badges where variant alone changes. Where text accompanies color it passes; where it doesn't, fails. **Severity: MEDIUM (WCAG 1.4.1)**.

Bonus/secondary (would round out a thorough remediation):
- Progress bars without ARIA progressbar role (DefiCard, wallet level progress).
- Raw enum strings in user-visible text (`pending_payment` etc. on parent food).
- `<pre>{JSON.stringify(...)}</pre>` rendered to end users (parent food page).
- `placeholder` used as label substitute in friends search and partner scanner.
